1. Архитектура проекта
Проект построен по классической слоистой архитектуре (Layered Architecture). Каждый слой отвечает за свое - роуты принимают запросы, middleware фильтруют их, модели работают с БД, сервисы содержат бизнес-логику.

1.1 Структура файлов
Файл	Слой	Ответственность
src/index.ts	Точка входа	Запуск Express, подключение роутов, инициализация БД
src/db.ts	База данных	Создание соединения с SQLite, инициализация таблиц
src/schema.ts	Схема БД	Описание таблиц через Drizzle ORM
src/models/user.ts	Модель	CRUD-операции над таблицей users
src/models/task.ts	Модель	CRUD-операции над таблицей tasks
src/services/tokenService.ts	Сервис	Вся логика JWT и refresh-токенов
src/middleware/auth.ts	Middleware	Проверка Access Token в заголовке
src/routes/auth.ts	Роутер	Маршруты: /register, /login, /refresh, /logout
src/routes/tasks.ts	Роутер	Маршруты CRUD для задач
src/types.ts	Типы	TypeScript-типы для всего проекта


1.2 Схема взаимодействия слоёв
Запрос от клиента проходит через несколько уровней:
Клиент (браузер/приложение)
       │  HTTP-запрос
       ▼
Express Router  ─── проверяет URL и метод
       │
       ▼
Middleware (auth.ts)  ─── проверяет JWT-токен
       │
       ▼
Route Handler  ─── валидирует входные данные
       │
       ▼
Model / Service  ─── бизнес-логика + работа с БД
       │
       ▼
SQLite (через Drizzle ORM)



2. База данных: schema.ts и db.ts
2.1 schema.ts - Описание таблиц
Drizzle ORM позволяет описывать структуру таблиц прямо в Typescript. Это называется "schema-first" подход: сначала описываем схему в коде, затем генерируем и применяем миграции.

Важно то, что пароли и refresh-токены никогда не хранятся в открытом виде. Только их bcrypt-хэши. Даже если базу данных украдут - злоумышленник не получит реальные данные.
